# SibyX 侵权取证 Checklist（发现抄袭 / 未授权商用后怎么办）

> 适用对象：SibyX（© 2026 sibyx & Litao Fan，采用自定义「SibyX 非商用许可证 / SibyX-NC」）。
> 本清单用于**发现有人复制代码并用于商业目的、或违反许可证分发**时，系统性地固定证据、确认侵权、依法维权。
> ⚠️ 这**不是法律意见**，只是操作手册；涉及诉讼/律师函请另行咨询知识产权律师。

---

## 0. 先记住三件事

1. **版权是自动产生的**——你写了代码、标了 `Copyright (c) 2026 sibyx & Litao Fan`，从创作完成那一刻起就受版权保护，**不需要**注册（登记只是增强举证力，见第 7 步）。
2. **SibyX-NC 只约束「他人」，不约束你自己**——你作为版权持有人，随时可以自己商用或另发授权。
3. **发现侵权后，第一动作是「固定证据」，不是「去骂对方」**——一旦打草惊蛇，对方可能删库、改指纹、跑路，证据就没了。

---

## 1. 别打草惊蛇（0 小时）

- ❌ 不要先在公开渠道 @ 对方、写 issue 辱骂、或在群里喊「有人抄我」。
- ❌ 不要先发邮件质问（会触发对方销毁证据）。
- ✅ 默默开始**固定证据**（下一步），把取证做成「铁证」后再行动。
- 内部建一个取证文件夹（本地 + 云备份各一份），所有材料按时间命名归档。

---

## 2. 固定「对方存在」的证据（0–24 小时）

目标：把对方站点/仓库**此刻的样子**原样冻存，证明「在某个时间点它长这样」。

- **网页归档（首选，免费、带时间戳）**
  - [archive.today](https://archive.today)（最稳，几乎永久、不删）
  - [Wayback Machine](https://web.archive.org/save)（提交存档）
  - 把对方**首页、疑似聊天页、关于页、定价/付费页**都各存一份，记下存档 URL。
- **整站截图 + 录屏**
  - 用浏览器「整页截图」保存；最好再录一段 10–30 秒操作录屏（打开页面→看源码→看控制台）。
  - 截图要带**系统时间/网址栏**（用浏览器自带截图或 Fireshot，保留地址栏）。
- **查看源代码并留档**
  - 在对方页面 `右键 → 查看网页源代码`（或 DevTools → Elements / Sources），把 `index.html` 与加载的 JS **另存为文件**（这是后面比对指纹的关键）。
- **区块链 / 可信时间戳（可选但推荐）**
  - 把上面的截图、源码文件做一次**哈希**（见第 7 步），再把哈希上链或提交「可信时间戳」服务，得到带时间的存在性证明。

> 实操建议：先做 archive.today + 截图，这两步 5 分钟搞定，已经能锁定「它曾经存在」。

---

## 3. 提取版权水印（核心！证明「这就是 SibyX 的代码」）

SibyX 在代码里埋了**隐蔽版权水印（项目指纹）**，用于一眼识别未授权复制。指纹值：

```
SIBYX-FP-2026-LF7A3D9C2B4E1580
```

在「对方站点」里按以下三处逐一搜索，**任一命中即坐实复制**：

| 位置 | 怎么搜 | 说明 |
|---|---|---|
| **页面源码** | `Ctrl+U` 打开源代码，搜 `sibyx-fingerprint` | 会看到 `<meta name="sibyx-fingerprint" content="SIBYX-FP-2026-LF7A3D9C2B4E1580" />` |
| **压缩后的 JS** | 在 DevTools → Sources 里搜 `SIBYX-FP-2026-LF7A3D9C2B4E1580` | terser 压缩后仍保留该字符串常量 |
| **运行时** | DevTools Console 输入 `__sibyx_fp` | 会返回 `'SIBYX-FP-2026-LF7A3D9C2B4E1580'`（不可枚举属性，但仍可读取） |
| **仓库/元数据** | 搜 `SIBYX-FP-2026-LF7A3D9C2B4E1580` 或 `sibyxFingerprint` | 若对方连 `package.json` 一起抄了，字段也会在 |

> 💡 这个指纹是**故意公开**的「金丝雀」：你自己在仓库里也搜得到它。发现别人站里有同样一串，复制关系就一清二楚。即使对方改了 UI、改了名字、换了 Logo，这串字符极难被顺手改掉——它就成了铁证。

---

## 4. 代码比对（证明「实质性相似」，而非巧合）

光有指纹还不够（对方可能辩称「只是碰巧」）。进一步做**实质性相似**比对：

- 把你仓库的 `app.js`（或压缩后的 `web3chat/app.js`）与对方抓下来的 JS 做 **diff**（VS Code / Beyond Compare / `diff`）。重点找：
  - **相同的独特字符串**：如中继命名空间 `gun.get('web3chat')`、特有的错误提示文案、UI 文案。
  - **相同的「Bug / 怪异写法」**：抄代码通常会把你的 bug 一起抄走，这是最强证据（独立重写不会撞同一个 bug）。
  - **相同的函数名 / 变量命名习惯**：如 `deriveAES`、`createEncryptedBackup`、`watchChannelMeta` 等。
  - **相同的非显而易见的结构**：如 E2EE 的 K 派发逻辑、虚拟滚动的代次防竞态写法。
- 对方若做了「改名混淆」，比对时关掉变量名只看**字符串常量与控制流结构**。
- 把 diff 结果截图/导出存证（标红相同段落）。

> 比对报告是给律师/平台看的关键材料，越结构化越好：列「相同点 1/2/3 + 对应行号 + 截图」。

---

## 5. 确认「是否商用」（决定维权力度与路径）

SibyX-NC 禁止商用。需确认对方是否在**商业使用**：

- 有**付费墙 / 订阅 / 按量计费**？
- 是 **SaaS / 售卖 / 内嵌进商业产品**？
- 页面有**广告 / 联盟 / 引流变现**？
- 背后是**公司主体**（用做其产品的一环）？
- 是否**移除你的版权声明 / LICENSE**（许可证明确要求保留署名）？

> 即便对方是「非商用但没署名」，也违约（许可证要求署名）；若是商用，则直接侵权 + 违约，可主张更强救济。

---

## 6. 收集「侵权主体」信息（决定去哪告、告谁）

- **域名 WHOIS**：查注册人 / 注册商 / 注册邮箱（很多会暴露真实身份或公司）。
- **公司主体**：对方若是产品，查其官网「关于/团队/备案/工商」信息。
- **托管商 & 部署平台**：
  - 前端托管（GitHub Pages / Vercel / Netlify / Cloudflare Pages / Render）→ 用于第 9 步平台投诉。
  - 代码仓库（GitHub / GitLab）→ 用于 DMCA / 仓库投诉。
  - 若有后端/中继，查其 IP/ASN/主机商。
- **联系线索**：对方留下的邮箱、社交账号、客服渠道（用于第 8 步发函，也用于证明「他收到了」）。
- 把以上信息整理成一张表，附截图。

---

## 7. 保存「时间戳」证据（让证据法庭/平台可用）

- **可信时间戳**：国内有官方授时机构（如 TSA）可对文件签发带时间的数字凭证；国际可用各类 timestamp 服务。
- **区块链存证**：把取证材料的哈希上链（以太坊/Flow 等），得到不可篡改的存在性证明。
- **公证处公证**：对关键页面做**网页保全公证**（最具法律效力，但收费）。重大侵权（涉商业金额大）才需要。
- **哈希留底**：对所有取证文件算 SHA-256，把哈希值记录在清单里，证明「文件未被篡改」。

> 一般纠纷用 archive.today + 哈希就够平台投诉；真要起诉再补公证。

---

## 8. 发函 / 通知（先礼后兵）

证据齐了，再行动。顺序建议：

1. **留存通知记录**：用**可留痕**的方式发（邮件 + 平台站内信 + 挂号信），证明「你通知过他」。
2. **停止侵权函（C&D）/ 律师函**：
   - 写明：你是 SibyX 版权人（© 2026 sibyx & Litao Fan）、对方具体侵权表现（附指纹命中 + diff）、要求（下架 / 停止商用 / 公开署名 / 删库）、期限（如 7–14 天）、保留追责权利。
   - 引用依据：`Copyright (c) 2026 sibyx & Litao Fan` + `SibyX-NC` 许可证（见仓库 `LICENSE`）。
   - 语气克制、事实清楚，**不威胁、不辱骂**（避免反被诉骚扰）。
3. **等待回应**：对方可能道歉下架、请求授权（你可选择收费授权——这正是双重许可模型的商业出口）、或沉默/对抗。

---

## 9. 平台投诉（让「第三方」帮你关掉它）

不同侵权载体走不同投诉通道：

- **GitHub / GitLab 仓库**：提交 **DMCA / 版权投诉**，要求下架侵权仓库（附你的版权证明 + 指纹证据 + 对方仓库链接）。
- **应用商店（App Store / Google Play / 小程序）**：以「侵权应用」申诉下架。
- **前端托管商（Vercel / Netlify / Cloudflare / Render）**：走其 **Abuse / DMCA** 通道举报。
- **广告 / 支付平台（若对方靠它变现）**：向 Google Ads / 支付宝 / 微信支付等举报「侵权变现」，可能直接切断其收入。
- **搜索引擎**：向 Google / 百度提交「侵权网页移除」请求（减轻扩散）。

> 平台投诉通常比诉讼快得多，是「低成本关停」的首选。

---

## 10. 持续监控（防止他换个地方再来）

- 把指纹 `SIBYX-FP-2026-LF7A3D9C2B4E1580` 设成**搜索引擎快讯**（Google Alerts / 百度资讯订阅），一旦网上出现这串字符就通知你。
- 定期（每月）在 **GitHub 代码搜索**、Google、Bing 搜这串字符。
- 在 SibyX 相关社区/论坛留意「长得像 SibyX 的项目」。
- 若发现新侵权，回到第 1 步循环。
