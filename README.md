## 设计要点（对应你的需求）

| 你的需求 | 本实现 |
|---|---|
| 不要 SIWE、简单登录 | ✅ 方案 A：**浏览器本地自动生成身份**，无密码、无钱包弹窗、无 SIWE |
| 记录只在本地的 | ✅ 聊天记录存于本浏览器 **IndexedDB**，默认不发给任何服务器 |
| 区块链技术 / 去中心化 | ✅ 频道消息用本地私钥 **ECDSA 签名**（可验证、防篡改）；**私聊用 ECDH 端到端加密**（AES-GCM，中继只见密文）；并可选 **GunDB P2P 同步**，多设备/多人可聊，但记录仍本地留存 |
| Web3 身份感 | ✅ 由签名公钥派生 `0x…` 风格地址，消息显示「✓ 已验证」 |

## 密钥模型（重要）

本地身份包含 **两对独立密钥**，互不混用：

- **签名密钥（ECDSA P-256）**：用于消息签名 / 验真，并派生 `0x…` 地址。
- **加密密钥（ECDH P-256）**：仅用于私聊端到端加密（ECDH 协商出 AES-GCM 共享密钥）。

> ⚠️ 不能用签名私钥去做 ECDH 密钥协商，WebCrypto 会直接报错。这是初版的一个实现坑，已修正。

## 私聊端到端加密原理

1. Alice 用 **自己的 ECDH 私钥 + Bob 的 ECDH 公钥** 派生出共享 AES 密钥；
2. 用该密钥 **AES-GCM 加密** 明文，并把密文 + 自己对密文的 **ECDSA 签名** 发出；
3. Bob 用 **自己的 ECDH 私钥 + Alice 的 ECDH 公钥** 派生出**同一个** AES 密钥，解密；
4. 中继/他人即使截获，因没有 Bob 的 ECDH 私钥，**无法还原明文**。

已用 Node WebCrypto 实测：跨方加解密一致 ✅、验签通过 ✅、错误密钥/中继无法解密 ✅。

## 运行方式

`crypto.subtle` 只在**安全上下文**可用，请用 localhost 或 https 打开（直接双击 `file://` 在部分浏览器也能用，但 localhost 最稳）。

```bash
# 前端（静态站）
cd chat4
python -m http.server 8080     # 或：npx serve .
# 浏览器打开 http://localhost:8080

# 可选：自建 GunDB 中继（让别人也能和你聊）
cd chat4/relay
npm install
node relay.js                  # 默认监听 8765，可设 PORT 环境变量
```

## 功能

1. **自动登录**：首次打开自动生成本地身份（双密钥），之后自动恢复。
2. **导出 / 导入身份**：点「导出」下载 `sibyx-identity.json`（含两对密钥与地址），换设备可「导入」恢复同一身份（同一地址）。
3. **本地留痕**：消息存 IndexedDB，刷新/重开都在。
4. **消息签名**：每条消息用签名密钥签名，渲染时验签并显示「✓ 已验证」，体现 Web3 可验证性。
5. **频道 / 私聊 / 好友**：
   - 频道：公开消息，签名但不加密，所有人可见可验。
   - 私聊：端到端加密（🔒 E2EE），内容仅双方可读。
   - 加好友两种方式：① 在频道里点对方消息的「＋好友」，**自动带上其签名公钥 + 加密公钥**；② 手动点「＋」，粘贴对方在「我的公钥」处**复制的「公钥卡片」**（JSON，含地址/签名公钥/加密公钥）。
   - 点好友即可进入私聊；对方须在频道发过消息或你已用其卡片加过，才有加密公钥可加密。
6. **去中心化同步（可选）**：勾选「去中心化同步」，并在「中继地址」填入你的中继 URL（**默认已填免费共享 demo 中继 `https://chat4hub-relay.onrender.com/gun`**：附件<1M，共享限速、有时会休眠，体验有限；联系开发者可使用专属中继提升体验感），消息经该中继 P2P 同步，不同设备/多人可同房间聊天；关闭则纯本地。
7. **群聊（加密成员制私人频道）**：创建者建 🔒 私有频道生成密钥 K；可「**➕ 邀请好友**」直接从好友列表拉人（端到端派发 K），或让对方申请、在「👥 成员」面板批准；会话列表显示成员数 `(n)`；创建者可「踢人」轮换 K。
8. **跨设备加密备份（恢复社交图谱）**：身份面板用「备份 ID + 口令」做 **AES-GCM 加密云备份**（口令经 PBKDF2 派生密钥，服务器只存密文、不可读），换设备「取回备份」即恢复账号 + 好友 + 频道 + 收藏；也可导出 `.sibyxkey` / 助记词本地搬运；附「立即同步」按钮与「上次同步」时间戳。
9. **长会话流畅 & 消息排序**：长聊天用虚拟滚动（只渲染可视区），上千条也不卡；消息**严格按收到时间排序，最新收到的永远在最下**（跨设备/断网重收也不再乱序）。

## 关于「去中心化」的说明

- **纯本地（默认）**：最贴合「记录只在本地的」。但单人单机，消息不与其他人互通。
- **GunDB 同步（可选）**：消息通过 P2P 中继节点转发，实现多人/多设备互通，这才是真正「去中心化」的部分。注意：开启后消息会出现在其他对等节点的缓存中——如果你要的是「连我的消息都不出本机」，请保持同步关闭。

## 部署：让别人也能打开并和你聊

本应用是纯前端 + 可选 Node 中继，**分两层部署**：

### ① 静态站（拿到分享 URL）
免费静态托管即可（不需要数据库/后端）：
- **GitHub Pages**（推荐，已附 `.github/workflows/pages.yml`）：把本目录推到 GitHub 仓库，仓库 Settings → Pages → Source 选 "GitHub Actions"，push 到 `main` 即自动发布，拿到 `https://<你>.github.io/<仓库>/`。
- 或 **Cloudflare Pages / Vercel**：直接连接仓库、根目录即发布目录，零配置。

> 注意：静态托管**只能放前端**，跑不了 GunDB 中继（那是 Node 服务）。

### 当前线上架构（Render Web Service，免费、无需绑卡）
线上 SibyX 实例（**`web3chat`**）托管**静态前端**（Express 独占，含 `/healthz` 健康检查）。客户端「中继地址」**默认指向免费共享 demo 中继** `https://chat4hub-relay.onrender.com/gun`：可接受小附件，但共享限速、空闲约 15 分钟会休眠（冷启需时），故免费用户体验有限——这**刻意**为之，正是后续推出**专属高速中继（付费订阅）**的动机：付费用户可获得 24/7 常驻、大附件支持与优先带宽的独立中继。

> 中继分级（freemium 模型）：**免费层** = 共享 `chat4hub-relay`（限速/会休眠，接受小附件）；**专属层（后续付费订阅）** = 独立高速中继（24/7、大附件、优先带宽）。当前默认即免费层；若要中继 24/7 常驻且不排队，可自托管（ngrok / VPS）或等待专属订阅上线。

> 若只想先上静态站（暂不要中继）：用 **New + → Static Site** 手动连同一仓库，Root Directory 填 `.`、Build Command 留空、Publish Directory 填 `.` 即可。

### ② GunDB 中继（让消息能跨人同步）
中继必须跑在能运行 Node 的地方。`relay/` 目录已备好（`package.json` / `relay.js` / `Procfile`）：
- **本机 + 内网穿透（最快验证）**：`cd relay && npm install && node relay.js`，再用 ngrok 等把 8765 暴露为公网：`ngrok http 8765`，把得到的 `https://xxxx.ngrok.io/gun` 填到客户端的「中继地址」。
- **免费 Node 托管（Render Web Service / Fly.io）**：连仓库 + 用 `Procfile`（`web: node relay.js`），或用上面的 **Blueprint**（`render.yaml`）一次把静态站和中继都部署了；拿到公网 URL 后填到「中继地址」。Render 免费层**无需绑卡**，但中继 Web 服务空闲 15 分钟会休眠（即当前默认免费中继 `https://chat4hub-relay.onrender.com/gun` 的行为）。
- **自有 VPS**：`node relay.js` 后台常驻（或 pm2 / systemd）。

### 朋友怎么连接你
1. 你把**静态站 URL** 发给朋友，他自己打开（或本地开同一份文件）；
2. 你们都勾选「去中心化同步」，并在「中继地址」填**同一个你的中继 URL**；
3. **频道**：双方加入同一个频道名（如 `global`），消息经中继互通；
4. **私聊**：朋友在频道点你消息的「＋好友」自动加你（带公钥），或你「复制我的公钥」发卡片给他粘贴；之后点好友进入 🔒 E2EE 私聊。

> 三个要点：
> - **存储始终本地**——即使开同步，聊天记录也只在各自身浏览器（IndexedDB）。
> - **别人能找到你 = 同步开着 + 中继可达**；关同步就是单机，谁都连不到你。
> - 当前 `gun.get('web3chat')` 为全局命名空间，频道名全世界共用；要做私人小圈子，请用独立中继 / 独立应用名。

## 近期变更（部署须知）

- **Gun 库改为本地自托管**：`index.html` 不再从外部 CDN（`gun.eco`）加载 Gun，而是引用仓库根目录的 `gun.js`（已随仓库提交）。原因：外部 CDN 极不稳定，会导致 `typeof Gun==='undefined'`、永远回退纯本地模式。部署时请确保 `gun.js` 一并上传到静态站根目录。
- **Render 静态站 `Publish Directory` 必须填 `.`**（根目录），不要填默认的 `public`——本仓库静态文件全在根目录，填 `public` 会构建失败（`Exited with status 1`）。
- **使用者昵称**：设置区可自设昵称，随消息广播给对方。
- **连接状态点**：顶栏徽章旁新增彩色圆点（灰=本地 / 橙=连接中·断开 / 绿=已连），直观显示联机状态。
- **聊天记录备份**：设置区新增「导出记录 / 导入记录」按钮，可把消息导出为 JSON 并再导入（按 id 合并去重）。
- **群聊（加密成员制私人频道）**：创建者建 🔒 私有频道自动生成密钥 K；「👥 成员」面板新增「**➕ 邀请好友**」可直接从好友列表拉人（端到端派发 K，对方无需申请）；会话列表私人频道显示成员数 `(n)`；创建者「踢人」会轮换 K（被踢者读不到新消息）。所有密钥交互沿用既有 E2EE 红线，服务端不可读。
- **跨设备加密备份（恢复社交图谱）**：身份面板支持「备份 ID + 口令」**加密云备份**（AES-GCM + PBKDF2，服务器只存密文、不可读）；换设备「取回备份」一次恢复账号 + 好友 + 频道 + 收藏，第二台设备不再是「裸身份」（早期版本只能恢复账号、看不到好友/频道入口）。也可导出 `.sibyxkey` / 助记词本地搬运；附「立即同步」按钮与「上次同步」时间戳。
- **长会话虚拟滚动**：会话超过 ~300 条自动启用窗口化渲染（只渲染可视区域），上千条消息也不卡；引用/跳转/搜索均统一走虚拟滚动定位。
- **消息排序修复**：消息**严格按「收到时间」升序、最新收到永远在最下**，比较器已在缺失时间戳时兜底、永不产生乱序 NaN；不同设备/断网重收都不再错位。
- **中继默认值更新（freemium 分层）**：客户端「中继地址」默认已改为**免费共享 demo 中继** `https://chat4hub-relay.onrender.com/gun`（接受小附件，但限速、空闲约 15 分钟会休眠，体验有限，刻意引导专属中继付费订阅）；**专属高速中继为后续付费订阅项**（24/7 常驻 + 大附件 + 优先带宽）。所有设备务必填**同一个**中继 URL，切勿多中继并列（异构中继会触发 Gun 异源图合并、截断字段）。

## 安全提示

- 身份私钥以 JWK 明文存于本浏览器 IndexedDB，**任何能访问此浏览器的人都能读取**。敏感场景请用「导出」备份后清除，或在隐私窗口使用。
- 本 Demo 未做 KYC / 传输层加密；若用他人公共中继请留意其可能缓存消息，**自建中继则数据仅经你自己节点**。跨设备同步走 WebSocket，建议最终用 https 托管前端。
- 若要对外发布成真实产品，按你之前的《钱包登录 + KYC 合规要点》，面向公众时通常需叠加 KYC 层。

## 匿名访问统计（隐私声明）

SibyX 使用 **cookieless 匿名统计**了解访问情况，以在不牺牲「隐私优先 / 无采集」定位的前提下支撑引流决策：

- **不设置 cookie、不使用 localStorage、不做设备指纹**；
- 访客 IP 在**服务端**经 `HMAC-SHA256` + **按日加盐** + 截断 16 位处理，**不可逆、且跨日不可关联**；
- 仅记录来源**域名**（不含完整链接）、渠道标记（如 `?ref=reddit`）与国家代码级聚合数据；
- 所有数据**仅用于聚合**，不做个体追踪；
- 统计开关在中继服务端（`UPSTASH_REDIS_*` 环境变量），未配置时统计自动停用，不影响主服务。

看板仅运营者可见：中继 `/admin` 需 `ADMIN_TOKEN` 令牌访问。
