# SibyX · 使用说明

本说明面向**普通使用者**（不是部署者）。你不需要懂区块链、不需要装钱包、不需要注册账号——打开网页就能聊。

> 应用场景：你想和朋友私密聊天，又不想把记录存到任何公司的服务器上。本应用把聊天记录只存在**你自己的浏览器**里，并通过可选的「去中心化同步」让你和朋友互通消息。
>
> 界面支持 **中文 / English** 两种语言，首次打开会**自动跟随你系统的语言**；你也可以随时手动切换。

---

## 一、怎么打开

聊天站是一个**网页应用**，部署在可联网的网址上。直接用浏览器打开部署好的地址即可使用——**无需安装、无需注册、无需密码**。打开后你的身份会自动生成（见第二节），立刻就能聊天。

> 它运行在你的浏览器里，所有密钥与聊天记录都留在本地（IndexedDB）。你只需记住那个网址，在任何设备的浏览器打开它、导入身份，就能回到自己的账号。

---

## 二、你的身份（自动生成）

- 首次打开，网页会在你浏览器里**自动生成一对本地身份**（含两把密钥），并显示一个 `0x…` 开头的地址，这就是你的「Web3 账号」。
- 这个身份**只存在你这台浏览器的本地存储（IndexedDB）**，不会上传到任何服务器。
- 刷新页面、关掉重开，身份都在，还是同一个地址。
- 你可以在「身份」面板设置**昵称**，昵称会随消息发给对方显示。
- **换设备 / 换浏览器想保留同一身份？** 在身份面板点「**导出身份**」下载一个 `sibyx-identity.json`（含你的密钥，请妥善保管）；在新设备打开网页后点「**导入身份**」选这个文件，就能恢复成同一个地址与好友关系。

> 🔑 **身份备份的作用**：它带走的是你的「账号（地址）+ 两把密钥」。换设备 / 遗失设备后，在新设备「**导入身份**」即可**恢复同一个地址、好友关系**，并且因为私钥回来了，你能**继续收发消息、解密私人频道与私聊**。
>
> **但要留意**：聊天记录本身只存在旧设备的浏览器本地（IndexedDB），**身份备份不含聊天内容**。所以——若旧设备还在（换机），请务必再用「**导出记录**」把历史一并带走，新设备「导入身份」+「导入记录」才能既恢复账号、又恢复全部对话（见第七节）；若设备已遗失且从未导出过记录，旧对话无法找回，但「导入身份」至少让你用回同一个账号、继续收发新消息。

> ⚠️ 私钥以明文存在本浏览器。任何能打开你浏览器的人都能读到。敏感场景请用「导出身份」备份后清除，或在隐私窗口使用。

> ⚠️⚠️ **谨慎「登出」与清理浏览器数据**：「登出」按钮会一次性清空本地的身份与全部聊天 / 好友记录（不可恢复）；在浏览器里「清除站点数据 / Cookie / IndexedDB」效果等同。二者都会让网页**重新生成一套全新身份（新地址）**——此时你对新身份而言已是「另一个人」：
> - **失去对旧消息的控制权**：旧身份发出的消息，其发送者地址与新地址不符，删除按钮不再显示，且即使强行广播删除，对方也会因「地址不匹配」而拒绝——你再也删不掉自己曾经发的那些话。
> - **旧私聊 / 私人频道密文也读不开**：没有旧密钥，历史加密内容无法解密。
>
> 🔑 **这恰是 SibyX 的魅力所在**：账号真正属于你（密钥在你手里，没有「找回密码」这种后门，也没有公司能替你重置）。代价是——**身份即钥匙，弄丢了就真没了**。所以：在登出或清理浏览器数据**之前**，请务必先「导出身份」+「导出记录」双备份；日后导入即可满血恢复同一身份与全部对话。

## 三、界面布局

- **电脑端（PC）**：左侧是「频道 / 好友」栏，右侧是聊天区，顶栏有语言切换和身份操作。
- **手机端**：默认只显示聊天区；点左上角 **☰** 按钮滑出左侧栏，点空白处（遮罩）或选好频道后自动收起。

---

## 四、公开频道（公开、可验证）

1. 在左侧「频道」输入框填一个频道名（如 `global`），点「**加入**」。
2. 频道里所有人发的消息都是**公开、带数字签名**的——每条消息旁会显示「✓ 已验证」，表示这条消息确实来自对应地址、没被篡改。
3. 你和朋友只要**加入同一个频道名**，并且都开启了同步（见第八节），就能互相看到消息。
4. 想移除某个频道？左侧频道列表里**每个非默认频道右侧都带一个 `✕` 删除按钮**，点它确认即可**删除该频道及其本地消息**（默认 `global` 频道受保护，不可删、始终保留）。

> 频道消息是公开的，不要发私密内容。私密内容请用私聊（第六节）或私人频道（第五节）。

---

## 五、私人频道（加密成员制）

如果你只想和**特定一群人**私密群聊，用私人频道。它像一个「加密的群」：消息**端到端加密**，只有被批准加入、持有频道密钥的成员才能解密。

- **创建**：在频道输入框填一个频道名，勾选「**🔒 私有（加密成员制）**」，点加入 → 你自动成为**创建者**，系统生成频道密钥 `K`。
- **加密效果**：频道内消息用 `K` 加密；未加入者看到的是密文（提示「🔒 加密消息，需批准加入后查看」），看不到明文。
- **邀请朋友加入**：朋友也填**同一个频道名**并勾选🔒私有加入，他们会看到密文和一个「**申请加入**」按钮；点击申请后，你（创建者/审批人）在频道头的「**👥 成员**」面板里会看到「待批准申请」，点「**批准**」→ 系统**通过端到端加密把密钥 K 自动发给他们**。
- **批准后**：对方即可读取并发送加密消息（未批准前不能发言）。
- **与私聊的区别**：私人频道是**多人的加密群**；私聊（第六节）只是两人之间。
- **管理成员**：创建者可「**踢人**」——系统更换密钥 `K`、重新分发给剩余成员，被踢者此后无法读取新消息（历史仍保留）。

> 私人频道的密钥通过端到端加密分发，中继服务器也无法得知明文。适合小团队、朋友群的私密讨论。

## 六、加好友 & 私聊（端到端加密）

**方式 A（最简单，需先在同一个频道里）**

1. 你和对方都加入同一个频道，对方在频道里发过一条消息；
2. 你点对方那条消息下方的「**＋好友**」按钮——系统会**自动带上他的两把公钥**，加好后就能私聊。

**方式 B（手动粘贴「公钥卡片」）**

1. 让对方在身份面板点「**复制我的公钥**」（复制的是一段 JSON「公钥卡片」），发给你；
2. 你点左侧「**＋好友**」，把那段 JSON 粘贴进去确认即可。

**私聊（🔒 E2EE）**：点左侧好友列表里某人进入私聊。私聊内容是**端到端加密**的：只有你和对方能解密看到明文，连中转服务器看到的也只是乱码密文。对方必须**已有你的加密公钥**才能给你发加密消息。

> 如果提示「对方缺少加密公钥」，按上方方式 A/B 补一下即可。

## 七、聊天记录备份：导出 / 导入

想把对话长期留存或迁移，可以备份聊天记录。**注意：这和「导出身份」是两回事。**

- **导出记录**：顶栏点「**导出记录**」→ 下载一个 JSON 备份文件，内含你的**全部聊天记录与频道列表**。
- **导入记录**：顶栏点「**导入记录**」→ 选择备份文件 → 系统按消息 id **合并去重**，并**并集频道名**（不会覆盖你已有的消息，是取并集）。
- **重要区别**：导出记录**只备份聊天内容，不含任何密钥**；导入后，私人频道 / 私聊里的加密内容仍需用对应的身份（导入身份或原有身份）才能解密。要完整迁移，请**同时用「导出身份」带走密钥**。

> 建议重要对话定期「导出记录」备份；换设备时，「导出身份」+「导出记录」一起用，才能在新设备上既恢复账号、又恢复全部对话。

## 八、去中心化同步（让消息能跨人/跨设备）

默认情况下，你的消息**只存在本机**，别人连不到你。要和朋友互通，需要开同步并指向同一个中转节点（中继）：

1. 顶栏找到「**去中心化同步**」开关，打开；
2. 在「**中继地址**」填你们约好的中继 URL（如 `https://chat4hub-relay.onrender.com/gun`）；
3. 你和朋友填**同一个**中继地址，加入**同一个**频道名，即可互通；加为好友后即可私聊或共建私人频道。

> 开启同步后，**聊天记录仍然只存在各自浏览器本地**（IndexedDB），只是消息会通过中继转发给对方。如果你要的是「连我的消息都不出本机」，请**保持同步关闭**。免费的公共中继节点多数已停用，请使用部署好的中继地址。

## 九、语言切换

- 顶栏右侧有「**中文 | EN**」两个按钮，点一下即时切换界面语言。
- 你的选择会被记住（下次打开还是你选的语言）。
- **首次打开**会自动匹配你操作系统/浏览器的语言：中文系统→中文，英文→EN，其他情况默认 EN。

## 十、常见问题

**Q：消息发不出去 / 朋友看不到我？**
A：检查是否打开了「去中心化同步」、中继地址是否填对且可达；确认你们频道名完全一致。纯本地模式（同步关着）本来就是单人单机，别人看不到你。

**Q：私聊显示「无法解密」？**
A：通常是公钥没对上。请重新用「＋好友」加对方（方式 A 最稳），确保双方都拿到了彼此的加密公钥。

**Q：换手机/电脑还能用原来的身份吗？**
A：能。用「导出身份」备份 `sibyx-identity.json`，在新设备「导入身份」即可恢复同一地址和好友关系。

**Q：私人频道别人能看到吗？**
A：不能。未批准加入的人只看到密文，看不到明文，也发不了消息。

**Q：记录会丢吗？**
A：只要不清浏览器缓存/本地存储、不换设备且没备份，记录就在。重要对话建议「导出记录」+「导出身份」一起备份。

**Q：这算真正的「去中心化」吗？**
A：纯本地模式最贴合「记录只在本地的」；开启 GunDB 同步后，消息经过 P2P 中继在多节点间转发，才算真正去中心化互通。两种模式都在你的掌控之中。

## 十一、匿名访问统计（隐私声明）

SibyX 使用 **cookieless 匿名统计**了解访问情况，以在不牺牲「隐私优先 / 无采集」定位的前提下支撑引流决策：

- **不设置 cookie、不使用 localStorage、不做设备指纹**；
- 你的 IP 在**服务端**经 `HMAC-SHA256` + **按日加盐** + 截断处理，**不可逆、且跨日不可关联**；
- 仅记录来源**域名**（不含完整链接）、渠道标记（如分享链接带 `?ref=reddit`）与国家代码级聚合数据；
- 所有数据**仅用于聚合**，不做个体追踪。

> 这保证了「你的密钥就是你自己、我们零采集」的承诺依然成立——统计层面我们看不到「是谁」，只看得到「今天大概有多少人、从哪些渠道来」。

## 十二、删除自己发的消息

- 你**自己发出**的每条消息，气泡右上角（昵称/时间那一行）都有一个红色 **✕** 按钮。
- 点 ✕ → 弹窗确认「删除后对所有人生效（仅能删除你自己发的消息，不可恢复）」→ 确认后：
  - **你自己这端**立即消失；
  - **所有人的那端也消失**（系统用你的签名广播了一条删除事件，对方端验签通过后才删，他人无法伪造删你的消息）。
- 适用范围：公开频道、私有加密频道、私聊——只要是**你自己发的**都能删。
- 边界：① 只能删自己发的（签名+地址校验防滥用）；② 这是**逻辑删除**而非密码学擦除（中继存储层可能残留底层数据，但前端不再显示）；③ 公开频道删后，别人若已截图/缓存仍持有——这点任何 IM 都避免不了；④ 需开启同步才会向他人广播，纯本地模式只删自己本机。

> 删除**不可恢复**，动手前想清楚。

## 十三、频道内昵称查重（自动消歧）

- 同一频道里，若**两个不同身份**用了**相同昵称**，系统会自动处理，保证你看到的名字不混淆：
  - 消息气泡的显示名，会自动在冲突昵称后加上**地址后 4 位**区分，例如 `小明·ab12` 与 `小明·cd34`；
  - 频道聊天区**顶部**出现黄色提示条：`⚠ 本频道有重名昵称「小明」，已自动用地址后4位区分显示。`；
  - 私有频道的**成员列表**也按同样规则加后缀区分。
- 查重范围：**本频道参与者**（在本频道发过言、或是私有频道成员即算），零额外网络流量；私聊（一对一）不查重。
- 新增消息到达会自动刷新消歧与提示。

> **为什么是「自动区分」而不是「禁止重名」？** SibyX 是去中心化、无中心服务器的架构，没有一台服务器能强行阻止两个人设一样的昵称。所以系统做的是**检测 + 视觉唯一**：任何人仍可设任意昵称，但同频道一旦重名，会自动用地址后缀把两个人区分开，你绝不会把 A 小明和 B 小明搞混。这是软提示方案，不设名时不做硬性拦截。

## 十四、数据备份与恢复（重要）

- SibyX 是去中心化架构：**你的全部聊天记录和身份密钥都只存在本机浏览器的 IndexedDB**，没有中心服务器替你保存，所以「备份 / 找回」完全靠你自己。
- **清除浏览器「缓存（图片和文件）」不会影响记录**——记录不在缓存里（客户端 Gun 也不使用浏览器缓存）。
- **清除「Cookie 和其他网站数据 / 站点数据」会一并清空 IndexedDB** → 聊天记录、身份私钥、私有频道密钥、所有设置全部丢失。
- 能否从中继（chat4hub-relay）找回来？
  - 中继虽把消息落盘（`./data`，radisk），但跑在 **Render 免费层，文件系统是临时性的**，每次休眠唤醒或重新部署都可能清空磁盘 → **不可靠，别当备份**。
  - 即便中继还有数据，清掉站点数据后你已变成「新身份」（私钥没了），私有频道 / 私聊的密文也解不开。
- 真正的保障 = 内置的**导出**功能：
  - **导出身份**：包含双私钥 + 地址的 JSON，是你账号的**唯一备份**。丢了 = 永久换身份，旧私聊谁也救不回。
  - **导出记录**：所有消息的 JSON 备份（按消息 id 合并去重、频道取并集，不含密钥）。
  - 换设备 / 清缓存后：导入身份 JSON → 恢复同一地址 → 能解密旧私聊 / 频道（中继还有数据时）；导入记录 JSON → 直接恢复历史。
- 三条实操建议：① 只清「缓存」没事；② 要清「站点数据」前，务必先点「导出身份」（+ 可选「导出记录」）把 JSON 存到本地安全处；③ 别依赖中继做备份，自己的导出文件才是王道。
